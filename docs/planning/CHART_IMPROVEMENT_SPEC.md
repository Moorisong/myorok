# 📊 차트 개선 기획 문서

## 대상: 최근 기록 차트 / 최근 강수·수액 차트

> 본 문서는 반려묘 병상일지 앱의 차트 가독성과 의료 전달력을 개선하기 위한 기획 문서이다.  
> **약/영양제 복용 차트는 본 개선 범위에서 제외한다.**

---

## 1. 개선 배경

현재 차트는 다음과 같은 한계가 있다.

- 숫자가 직관적으로 보이지 않음
- 이벤트성 데이터가 연속 데이터처럼 오해될 소지가 있음
- 병원에서 빠르게 읽기 어려움
- 전문 의료 기록처럼 보이지 않음

본 개선의 목표는  
**"그래프를 해석하지 않아도 수치가 바로 읽히는 차트"**이다.

---

## 2. 개선 대상 차트

### 포함
- 최근 기록 차트 (배변 / 설사 / 구토)
- 최근 강수 / 수액 차트

### 제외
- 약 복용 차트
- 영양제 복용 차트

---

## 3. 최근 기록 차트 개선안

### 3.1 차트 목적
- 날짜별 증상 **발생 횟수**를 즉시 파악
- 의료진이 질문 없이 이해 가능하도록 구성

---

### 3.2 데이터 성격 정의
- 하루 1회 기록
- **이산 데이터 (연속 아님)**
- 단위: `회`

---

### 3.3 차트 타입
**Dot Chart (점 중심 차트)**

- 선(line)은 기본적으로 사용하지 않음
- 점(dot)이 데이터의 본체 역할

---

### 3.4 표현 규칙

| 항목 | 규칙 |
|----|----|
| X축 | 날짜 |
| Y축 | 횟수 (정수만) |
| 데이터 표시 | 점(dot) |
| 수치 표시 | 점 바로 위에 숫자 고정 표시 |
| 단위 | `회` |
| 데이터 없는 날 | 공백 (0으로 연결하지 않음) |

---

### 3.5 증상 분리 원칙
- 한 차트에 여러 증상을 혼합하지 않는다.
- 각 증상은 **독립 차트**로 표시한다.

```
[배변 횟수 차트]
[설사 횟수 차트]
[구토 횟수 차트]
```

---

### 3.6 의료 전달 관점 기준
- "며칠에 몇 번 있었는지"가 즉시 보일 것
- 추세보다 **정확한 수치 전달 우선**

---

## 4. 최근 강수 / 수액 차트 개선안

### 4.1 차트 목적
- 강수/수액 **투여량(ml)**을 날짜별로 명확히 전달
- 연속 투여로 오해되지 않도록 설계

---

### 4.2 데이터 성격 정의
- 이벤트성 데이터
- 연속 데이터 ❌
- 단위: `ml`

---

### 4.3 차트 타입
**Bar Chart (막대 차트)**

- 선(line) 사용 금지
- 날짜별 독립 막대

---

### 4.4 표현 규칙

| 항목 | 규칙 |
|----|----|
| 기본 표현 | 세로 막대 |
| 연결 | 절대 연결하지 않음 |
| 수치 표시 | 막대 상단에 ml 숫자 고정 |
| 단위 | `ml` |
| 날짜 내 다회 투여 | 합산하여 1개 막대 표시 |

---

### 4.5 다회 투여 표시 예시
```
총 250ml (2회)
```

---

### 4.6 투여 타입 구분
- 시각적 구분만 사용 (색 또는 아이콘)
  - 💧 강수
  - 💉 피하 수액 (subcutaneous)
  - 🧪 정맥 수액 (iv)

---

## 5. 수치 가독성 공통 규칙 (중요)

모든 최근 차트는 아래 규칙을 따른다.

- 숫자는 **항상 화면에 노출**
- 툴팁은 보조 수단
- 소수점 사용 금지
- 단위 생략 금지

### 예시
```
● 3회
█ 120ml
```

---

## 6. 디자인 및 UX 공통 원칙

| 항목 | 원칙 |
|----|----|
| 애니메이션 | 최소화 |
| 그리드 | 최소 |
| Y축 자동 스케일 | 사용하지 않음 |
| 데이터 밀도 | 최근 7~15일 기준 |
| 의료 오해 가능성 | 0을 목표 |

---

## 7. 최종 기획 한 줄 요약

> 최근 차트는 "예쁜 추이 그래프"가 아니라  
> **병원에 바로 보여줄 수 있는 의료 기록 시각화**를 목표로 한다.
