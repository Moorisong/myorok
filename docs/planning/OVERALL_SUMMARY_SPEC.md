# 📊 Overall Summary (전체 요약 카드) 기획 문서

## 개요
"전체(All)" 기간은 이벤트 기반 차트(Dot/Bar)와 구조적으로 맞지 않기 때문에  
기존 차트를 제거하고 **숫자 중심의 요약 통계 카드**로 대체한다.

이 화면은 **장기 추이를 그리는 곳이 아니라**,  
병원에서 "이 아이의 전체 관리 상태"를 빠르게 설명하기 위한 용도이다.

---

## 적용 위치
- Charts 탭 → Summary
- 기간 선택:
  - 15일 / 1개월 / 3개월 / **전체(요약)**

---

## 전체(요약) 화면 구성

### ❌ 제거
- 모든 그래프/차트
- Dot / Bar / Timeline 시각화

### ✅ 유지
- 숫자
- 단위
- 명확한 기간 정보

---

## 요약 카드 구성 (Stat Cards)

### 1️⃣ 기록 기간 카드
```
기록 기간
2024.03.12 ~ 2025.09.30
(총 568일)
```

---

### 2️⃣ 증상 요약 카드
```
증상 요약
구토: 42회
설사: 18일
평균 배변: 1.3회 / 일
```

#### 계산 기준
- 구토: 전체 기간 누적 횟수
- 설사: 설사 발생 "일수"
- 배변: (전체 배변 횟수 ÷ 기록 일수)

---

### 3️⃣ 강수 / 수액 요약 카드
```
강수 / 수액 요약
강수: 4,200 ml
수액: 3,800 ml
총 투여량: 8,000 ml
```

---

### 4️⃣ 관리 밀도 카드 (선택)
```
관리 요약
기록된 날: 412일
기록률: 72%
```

> 병원에서 "관리 잘 된 아이"를 수치로 보여주는 용도

---

## UX 원칙

- 차트 ❌
- 스크롤 최소
- 숫자는 항상 화면에 노출
- 소수점 최소화 (배변 평균만 허용)
- 툴팁 사용하지 않음

---

## 의료 전달 관점 목표

- "이 아이는 전반적으로 이렇게 관리됐다"를 10초 안에 전달
- 긴 기간에서도 가독성 유지
- 이벤트성 데이터의 왜곡 방지

---

## 구현 가이드 (summary.tsx)

### 구조
- 기간이 `ALL`일 경우:
  - Chart 컴포넌트 렌더링 ❌
  - `OverallSummaryCards` 컴포넌트 렌더링 ⭕

### 데이터 처리
- 모든 값은 사전 집계
- 날짜 루프 렌더링 없음
- 성능 영향 최소

---

## 한 줄 요약
> "전체"는 차트가 아닌 **의료 요약 리포트**로 보여준다.
