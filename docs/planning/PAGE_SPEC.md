# 페이지 라우터 & UI/UX 명세 (MVP)

> **모바일 앱(React Native + Expo)** 기준 페이지 구조, 라우팅, UI/UX 목적 정의

---

## 1. 전체 라우팅 구조

```
/ (Home - 오늘 탭)
├─ /today (오늘 기록 편집)
├─ /calendar (캘린더 - 조회 전용)
├─ /records (기록 카테고리)
│   ├─ /food
│   ├─ /medicine
│   ├─ /hospital
│   ├─ /fluid
│   └─ /custom-metrics
├─ /charts
│   ├─ /summary
│   └─ /custom
├─ /pro
├─ /settings
└─ /about
```

**탭 네비게이션:**
```
[ 오늘 📋 ] [ 캘린더 📅 ] [ 차트 📊 ] [ 설정 ⚙️ ]
```

> 탭 네비게이션 + 스택 네비게이션 혼합 사용

---

## 2. 공통 UX 원칙

| 원칙 | 설명 |
|------|------|
| 입력 방식 | "한 번에 입력" ❌ → "확인할 때마다 +1" ✅ |
| 입력 최소화 | 버튼 중심 인터랙션 |
| 과거 데이터 | 숨기지 않고 흐리게 표시 |
| 다묘 지원 | 상단 고양이 선택 드롭다운 |
| 최종 목표 | 병원에서 바로 보여줄 수 있는 화면 구조 |

### 앱 진입 UX ⚠️ 중요

| 상황 | 동작 |
|------|------|
| 앱 실행 시 | 무조건 **오늘 기록 화면** 진입 |
| 고양이 1마리 | 바로 해당 고양이 기록 화면 |
| 고양이 2마리 이상 | 마지막 선택 고양이 기록 화면 (AsyncStorage 저장) |
| 과거 조회 | 의식적으로 진입해야 함 |

> 이 UX가 기록 유지율을 좌우함

## 3. 페이지별 명세

### 3.1 Home (`/`)

| 항목 | 내용 |
|------|------|
| 역할 | 오늘 상태 한눈 요약 + 빠른 기록 진입 |
| 구성 | **고양이 선택 드롭다운**, 오늘 날짜, 소변/배변/묽은 변/구토 요약, 약 복용 여부, "오늘 기록하기" 버튼 |
| 다묘 | 상단에 고양이 선택 UI, 선택 시 즉시 데이터 갱신 |

---

### 3.2 Today (`/today`)

| 항목 | 내용 |
|------|------|
| 역할 | 하루 누적 입력의 중심 화면 |
| 기본값 | 모든 카운트 0 |
| UX | 버튼 누를 때마다 즉시 +1 |
| 다묘 | **상단 고양이 선택**, 선택한 고양이 기준 입력 |

**입력 항목:**
| 항목 | 입력 방식 | 비고 |
|------|----------|------|
| 소변 | +1 버튼 | - |
| 배변 | +1 버튼 | 묽은 변 토글 (분리 카운트) |
| 구토 | +1 버튼 | 색상 라벨 선택 |
| 강수 | +추가 | ml 단위 리스트 누적 (강제 급수) |
| 특이사항 | 메모 | 자유 입력 |

---

### 3.3 Records 허브 (`/records`)

| 역할 | 기록 카테고리 진입점 |
|------|---------------------|

**메뉴:**
- 기본 컨디션
- 사료 기호성
- 약 / 영양제
- 병원 기록
- 수액 기록
- 커스텀 수치

---

### 3.4 사료 기호성 (`/records/food`)

| 항목 | 값 |
|------|-----|
| 사료 타입 | 건사료 / 캔 / 기타 |
| 기호성 | 잘먹음 / 보통 / 거부 |
| 코멘트 | 자유 메모 |

---

### 3.5 약 / 영양제 (`/records/medicine`)

- 유저 입력 → 라벨 생성
- **[Today] 탭에서 인라인 추가 가능**
- 체크박스 방식
- 복용 여부만 기록 (용량 ❌)

---

### 3.6 병원 기록 (`/records/hospital`)

| 항목 | 비고 |
|------|------|
| 방문 날짜 | 필수 |
| 진단/처방 요약 | 텍스트 |
| 자유 메모 | - |

> 병원용 차트와 연계 전제

---

### 3.7 수액 기록 (`/records/fluid`)

| 항목 | 비고 |
|------|------|
| 날짜 | 필수 |
| 수액 종류 | 선택/입력 |
| 용량 | 숫자 |
| 반응/메모 | 자유 입력 |

---

### 3.8 커스텀 수치 (`/records/custom-metrics`)

| 항목 | 내용 |
|------|------|
| 목적 | 장기 추적할 수치 정의 및 값 입력 |
| 수치명 | 직접 입력 (라벨 생성) |
| 값 | 숫자 입력 |
| 메모 | 선택 |
| 차트 | ❌ 이 화면에서 표시 안 함 |

---

## 4. 차트 페이지

### 4.1 요약 차트 (`/charts/summary`)

| 항목 | 내용 |
|------|------|
| 목적 | 병원 제시용 핵심 화면 |
| 구성 | **고양이 선택**, 증상 변화 추이 (배변/묽은 변/구토), 약 복용 기간 Bar 오버레이, 날짜 기준 스크롤 |
| 다묘 | 선택한 고양이의 차트만 표시 |

---

### 4.2 커스텀 차트 (`/charts/custom`)

| 항목 | 내용 |
|------|------|
| 목적 | 사용자 정의 수치의 변화 추이 시각화 |
| UX | 수치 선택 → 기간 선택 |

**무료 vs 유료:**
| 구분 | 표시 |
|------|------|
| 무료 | 최근 15일 실선, 과거는 흐림/점선 |
| 유료 | 전체 기간 표시 |

---

## 5. Pro 유도 페이지 (`/pro`)

| 항목 | 내용 |
|------|------|
| 역할 | 유료 기능 설명, 과거 데이터 접근 해제 안내 |
| 강조 | "이미 데이터는 저장되어 있습니다" |
| 예시 | 병원 차트 예시 노출 |

## 6. 설정 (`/settings`)

- **고양이 관리** (추가/편집/삭제)
- Pro 업그레이드
- 앱 정보
- 개인정보 처리방침
- 이용약관
- 초기화(전체 삭제) 기능

### 6.1 고양이 관리

| 기능 | 설명 |
|------|------|
| 추가 | 이름 입력, UUID 자동 생성 |
| 편집 | 이름 수정 가능 |
| 삭제 | 소프트 삭제 (`deletedAt`), 복원 가능 |
| 목록 | 활성 고양이만 표시 |
