# Web Landing Page Agent Reference

## WEB_LANDING_PAGE_SPEC (WEB_LANDING_PAGE_SPEC.md)
# WEB_LANDING_PAGE_SPEC

## 1. 목적 (Purpose)
- **한 줄 목표**: 방문자가 3초 안에 앱을 이해하고, 30초 안에 설치 버튼을 클릭하도록 유도함.
- **역할**: 서비스 기능 제공이 아닌, '앱 설치 유도' 전용 랜딩 페이지.
- **제외 사항**: 로그인, 기능 제공, 기록 저장, 서비스 체험 없음.
- **핵심 가치**: 공감 + 신뢰 + 안심 → 설치.

## 2. 서비스 정의 및 타겟 (Definition & Target)
- **서비스 정의**: 아픈 고양이를 위한 기록·케어 관리 앱.
- **핵심 타겟**:
    - 질병 치료 중인 고양이를 키우는 보호자.
    - 만성질환(신장, 당뇨, 심장 등) 묘주.
    - 병원 방문이 잦고 투약/증상 기록이 필수인 보호자.
- **사용자 감정**: 불안, 죄책감, 기록의 필요성은 느끼나 지속이 어려움.
- **니즈**: "기억이 아니라 기록이 필요하다." (우리 고양이를 더 잘 돌보고 싶은 마음).

## 3. 페이지 구조 및 UX (Page Structure & UX)
**스크롤형 원페이지 구조**: Hero → 공감 → 해결 → 기능 → 신뢰 → 설치 CTA 반복

### 3.1 Hero 영역 (첫 화면)
- **목표**: 3초 안에 이해 + 감정 자극.
- **구성**:
    - 헤드라인: "아픈 고양이를 위한  
      기록 케어 앱"
    - 서브텍스트: "투약 · 증상 · 병원 기록을  
      한 곳에서 쉽고 빠르게"
    - CTA 버튼: "👉 묘록 설치하기" (Google Play 링크 이동)

### 3.2 공감 섹션
- **메시지**:
    - "약 먹인 시간 기억하시나요?"
    - "증상이 언제 시작됐는지 떠오르시나요?"
    - "병원에서 설명하기 어려웠나요?"
- **강조**: "보호자는 기억해야 할 것이 너무 많습니다."

### 3.3 해결 제시
- **메시지**: 묘록은 **아픈 고양이 케어 기록을 쉽게 만드는 앱**입니다.

### 3.4 핵심 기능 소개
1.  **투약 기록**: 약 이름/시간/메모 기록. "약 언제 먹였지?" 스트레스 감소.
2.  **증상 기록**: 사진+메모. 작은 변화도 놓치지 않음. "작은 변화가 큰 신호가 됩니다."
3.  **병원 방문 준비**: 기록 통합 관리. "병원 갈 때 가장 든든한 기록."

### 3.5 차별점 (Why 묘록?)
- **기존**: 사진 앱(흩어짐), 메모 앱(검색 어려움), 기억(잊어버림).
- **묘록**: 아픈 고양이 케어 전용 앱. "**기억이 아닌 기록으로 돌보기**".

### 3.6 안심 섹션 (신뢰)
- **메시지**: 누구나 쉽게 시작, 복잡한 설정 없음, 부담 없는 기록. "기록은 보호자를 안심시킵니다."

### 3.7 CTA (Call To Action) - 반복
- **문구**: "오늘부터 기록하면 내일이 훨씬 편해집니다."
- **버튼**: "👉 묘록 설치하기"

### 3.8 Footer
- 포함: 플레이스토어 링크, 개인정보처리방침, 문의 이메일.

## 4. 디자인 및 구현 전략 (Design & Implementation)
- **디자인 톤**: 따뜻함, 신뢰감, 안정감, 파스텔 톤. (의료적 차가움 지양).
- **참고 문서**: `docs/planning/DESIGN_SYSTEM.md` (컬러 팔레트 및 폰트 참고하되, 랜딩 페이지의 따뜻한 톤에 맞춰 조정 가능).
- **기술 스택**: Next.js 또는 React (기존 `apps/web` 환경 활용).
- **반응형**: 모바일 최적화 필수 (앱 설치 유도 특성상 모바일 접속 비중 높음).
- **SEO**:
    - Title: 묘록 - 아픈 고양이를 위한 기록/케어 앱
    - Meta Description: 투약, 증상, 병원 기록을 한 곳에서. 만성질환 고양이 필수 앱.
- **링크**:
    - Play Store: `https://play.google.com/store/apps/details?id=com.myorok.app&pcampaignid=web_share`

## 5. KPI
- 설치 버튼 클릭률 (CTR).
- 플레이스토어 이동률.

## AI 작업 지침

### 목적
- '묘록' 앱 설치 유도를 위한 웹 랜딩 페이지(Web Landing Page) 구현 및 유지보수.
- 사용자에게 공감과 신뢰를 주어 빠르고 자연스럽게 앱 설치로 연결.

### 작업 단계
1.  **환경 확인 (Setup)**: 
    - `apps/web` 디렉토리 내 Next.js 프로젝트 설정 확인 (없으면 생성).
    - `docs/planning/DESIGN_SYSTEM.md`를 참고하여 기본적인 테마 설정 (Tailwind CSS 설정 등).
2.  **구조 구현 (Structure)**: 
    - `WEB_LANDING_PAGE_SPEC.md`의 8개 섹션(Hero ~ Footer)을 컴포넌트 단위로 분리하여 구현.
    - 스크롤 시 자연스러운 전환 애니메이션(Framer Motion 등 활용) 고려.
3.  **디자인 & 콘텐츠 (Design & Content)**:
    - **톤앤매너**: 따뜻하고 부드러운 파스텔 톤.
    - **이미지**: `generate_image` 툴을 활용하여 각 섹션에 맞는 따뜻한 일러스트 생성 및 적용.
    - **텍스트**: Spec에 정의된 문구를 정확히 적용하며 가독성 확보.
4.  **기능 및 SEO (Function & SEO)**:
    - 모든 "설치하기" 버튼에 Google Play Store 링크 연결.
    - 메타태그(Title, Description, OG Tag) 완벽 적용.
    - 모바일 뷰포트 최적화 확인.
5.  **검증 (QC)**:
    - 모바일 기기에서의 레이아웃 깨짐 확인.
    - 로딩 속도 최적화 (이미지 압축 등).

### 주의사항
- **단순함 유지**: 불필요한 기능이나 복잡한 메뉴바는 제거한다.
- **모바일 우선**: 데스크탑보다 모바일 사용자가 주 타겟임을 명심하고 모바일 디자인에 우선순위를 둔다.
- **감성적 접근**: 기술적인 설명보다는 '집사의 마음'을 이해하는 문구와 디자인을 사용한다.
